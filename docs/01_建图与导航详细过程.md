# VGP-Nav 建图与导航 —— 详细过程文档

> 复现论文 **VGP-Nav: Metric-Aware Visual Geometric Perception for Robot Navigation**
> (Pan et al., arXiv 2606.09268)。纯单目 RGB,同时做**度量定位**与**度量障碍感知**,
> 核心是用**地平面几何**把单目重建锚定到真实物理尺度。
>
> 本文档讲清"从原始数据到可导航地图、再到在线定位导航"的**完整过程与原理**;
> 配套的逐条命令见 [`02_代码执行指南.md`](./02_代码执行指南.md)。

---

## 0. 一页总览

VGP-Nav 分**离线建图**和**在线导航**两大阶段:

```
┌─────────────────────────── 离线建图 (一次性) ───────────────────────────┐
│ 原始4路鱼眼视频(Mapping_C8)                                              │
│   │ ① 鱼眼→针孔去畸变 (MEI 模型)                                         │
│   │ ② VGGT 滑窗链式重建 + 逐窗重力对齐 + 统一尺度 + 刚性拼接 → 上尺度轨迹 │
│   │ ③ 全局重力对齐 + 地面峰 + 相机高1.3m → 度量尺度锚定 (地面 z=0)       │
│   │ ④ 划分 DB(每3帧1个) / query(留出评测) + 算 VPR 描述子               │
│   │ ⑤ 4路环视 VPR 描述子 → 4路交叉回环检测 → SE(2) 位姿图优化(PGO) 消漂移│
│   │ ⑥ 用对齐后的位姿重建 4路环视融合点云 (360°度量点云)                  │
│   │ ⑦ 点云 → 2D 占据栅格 (稳健判障 + 轨迹走廊强制可通行)                 │
│   └→ 产物: 带位姿的 DB + 全局点云 + 占据地图 (即"最终用于导航的地图")     │
└──────────────────────────────────────────────────────────────────────┘
┌─────────────────────────── 在线导航 (每帧 query) ───────────────────────┐
│ query 4路环视图                                                          │
│   │ ① VPR 检索 (4路取最大相似, 抗朝向差) → top-N 候选                     │
│   │ ② 几何感知检索 (Algorithm 1): 空间离群剔除 + 多样性选择 → k=10 参考  │
│   │ ③ VGGT([query]+k refs) → 相对位姿 + 稠密点                           │
│   │ ④ 加权运动平均 (旋转共识 + IRLS 射线三角化) → 世界位姿 T_wc (定位)   │
│   │ ⑤ 地面锚定恢复本帧尺度 → 度量点云                                    │
│   └→ 占据图 + A* → 无碰撞路径 (导航)                                     │
└──────────────────────────────────────────────────────────────────────┘
```

**与论文的关键差异(工程取舍)**:
- VPR 用 **SelaVPR++**(DINOv2-large, 4096维)或 DINOv2,而非论文的 NetVLAD。
- 数据无真值位姿,用 **VGGT 滑窗重建 + 1.3m 地面锚定自建伪位姿库**作为参考 DB。
- 针对真实办公室数据的**单目长序列漂移**,额外做了 **4路环视回环 PGO**(本复现的关键加强,详见 §3.5)。

## 1. 数据与坐标系约定

### 1.1 原始数据 (Mapping_C8)
- 采集自 DEEPROUTE 办公室一层,**626 帧 × 4 路鱼眼相机** = 2504 张 jpg。
- 文件命名 `{timestamp}_camera_{N}.jpg`,N∈{1,2,3,4}。同一 timestamp 的 4 张是同步的环视。
- **无真值位姿**(伪 GT 由建图自产)。

### 1.2 4 鱼眼相机的物理布局
- MEI 鱼眼模型,4 路环视覆盖 360°。相对车体的方位(用于可视化4环视布局):
  - `camera_1` = 前左,`camera_2` = 前右,`camera_3` = 后右,`camera_4` = 后左。
- 内参格式 `[xi, fx, fy, cx, cy]`(xi 是 MEI 球面参数),畸变 `[k1,k2,p1,p2]`。
- **定位主锚是 camera_1**(单目度量定位);cam2/3/4 用于补 360° 检索/几何/感知。

### 1.3 坐标系
| 记号 | 含义 |
|---|---|
| 相机系 | OpenCV 约定:x 右、y 下、z 前 |
| `world_v` | VGGT 自定义参考系(首帧相机为单位阵),**尺度自由** |
| `extri` (3×4) | VGGT 输出 `[R\|t]`,表示 `world_v → cam`(X_cam = R·X_v + t) |
| `T_wc` (4×4) | `cam → world`(度量世界系),相机中心 C = -Rᵀt |
| 世界系 | 重力对齐(z 向上)、地面在 z=0、米制;x-y 原点平移到首帧 |

## 2. 离线建图 —— 步骤①②③④:从原始数据到带位姿数据库

> 代码:`vgpnav/database.py` 的 `build_database()` + `_chain_trajectory()`;入口脚本 `scripts/build_database.py`。

### 2.1 步骤① 鱼眼 → 针孔去畸变 (`vgpnav/undistort.py`)
单目度量重建 VGGT 需要针孔图。`PinholeUndistorter` 把 MEI 鱼眼重采样为针孔:
- **MEI 投影** `_project(X)`:单位化 → 球面投影 `(xu,yu)=xs[:2]/(xs[2]+xi)` → 径向畸变 `1+k1·r²+k2·r⁴` → 切向畸变 → 鱼眼像素 `u=fx·xd+cx`。
- **逆映射表** `_build_maps()`:对每个目标针孔像素,反算其在鱼眼图中的来源坐标(可选俯仰 `pitch_down` 让相机多看地面),存 `map_x/map_y`,用 `cv2.remap` 双线性重采样。
- **虚拟针孔内参**:`f_v=(W/2)/tan(hfov/2)`,主点在图像中心。默认输出 640×480、HFOV 110°。

### 2.2 步骤② VGGT 滑窗链式重建 (`_chain_trajectory`,两遍)
单目 VGGT 每个窗口尺度独立模糊,仅末尾一次全局锚定无法消除逐窗尺度差累积成的漂移(重建地面随轨迹抬升、整体被撑大、重访区错位)。故采用**两遍**:

**Pass 1(逐窗,VGGT 只跑一遍并缓存):**
对每个滑窗(`chunk_size=10` 帧,相邻重叠 `chunk_overlap=6`):
1. `VGGTRunner.infer(imgs)` → 每帧 `extri`(world_v→cam)、`world_points`(H,W,3)、`depth_conf`。
2. **逐窗重力对齐**:由窗内各帧的相机 y 轴(世界系 = R_wc·[0,1,0])平均估重力向下 `g_down`,`Rg = rotation_align(g_down, [0,0,-1])`,把窗内位姿和点云都旋到重力对齐。
3. **逐窗原始尺度**:`anchor_map(centers, points, 1.3)` 用"地面峰 + 相机中位高 / 1.3m"估该窗尺度,缓存。

**统一稳健尺度:** `s_uni = median(各窗原始尺度)`。逐窗地面估计噪声大(杂乱区误判地面→虚高),各自生效会把地图撑爆;VGGT 各窗原生尺度近似一致,故用中位数消噪、保证窗间一致(绝对尺度留给末尾全局锚定)。

**Pass 2(应用统一尺度 + 刚性拼接):**
- 每窗位姿平移、点云乘 `s_uni`。
- 与已放置的全局帧的**共享帧**做 `align_sim3_from_poses(..., fix_scale=True)`(只求 R,t,尺度固定为1),把当前窗刚性拼到全局。
- 累积全局位姿 `global_T` 和点云。

→ 得到 up-to-scale 的全局轨迹位姿 + 稠密点云。

### 2.3 步骤③ 全局重力对齐 + 地面锚定度量尺度
1. **全局重力对齐**:所有帧相机 y 轴平均估 `g_down`,`R_align` 旋到 z 向上,位姿/点云一起转。
2. **单次全局地面锚定** `anchor_map`:
   - `find_ground_peak(z)`:沿重力轴对点高度直方图,在**下半部分**(`low_frac=0.6`)取主峰作地面高度 `ground_z`。
   - `scale = camera_height(1.3) / (median(相机z) - ground_z)`,`z_shift = -ground_z·scale`(地面移到 z=0)。
3. 应用 `X_w = scale·X + [0,0,z_shift]`,再把 x-y 原点平移到首帧。
→ 全图进入**米制、重力对齐、地面 z=0** 的世界系。`meta.json` 记录此 `scale`。

### 2.4 步骤④ DB / query 划分 + VPR 描述子
- 轨迹帧 `pos % db_sub(=3) == 0` → **DB 节点(209 个)**;其余里均匀抽 `n_query(=25)` 作评测 query。
- 对每个 DB 帧的 camera_1 去畸变图算 VPR 描述子(`vgpnav/vpr.py`,SelaVPR++ 4096维,L2 归一化)。
- **产物**(`outputs/db/`):`trajectory.npz`(626帧位姿)、`db.npz`(209节点位姿+朝向+描述子)、`query.npz`(25 伪GT)、`global_points.npz`(下采样点云)、`meta.json`。

## 3. 离线建图 —— 步骤⑤⑥⑦:消漂移、融合点云、生成最终占据地图

### 3.1 为什么还需要 PGO(漂移割裂问题)
单目 VGGT 长序列建图**全局漂移但局部一致**。机器人绕行/重访时,同一物理位置的两趟重建会**错位**(同一面墙裂成两个副本),反映到占据图就是**虚构障碍**(把走廊误判成障碍、把房间内部填满)。诊断脚本 `detect_split.py` 按"生成关键帧时间(kt)"给点云着色,割裂区表现为**双色错开**。

### 3.2 步骤⑤-a 4 路环视 VPR 描述子 (`scripts/compute_4cam_desc.py`)
- 对全部 626 帧的 **4 路**都算 SelaVPR 描述子,存 `allframe_desc_4cam.npy`,形状 `(626, 4, D)`。
- **动机(本复现的关键洞察)**:机器人重访常**反向**经过同一地点。此时 camera_1(单目)看到的景象相反、VPR 相似度很低(实测帧 621 vs 501 仅 **0.32**),回环检测会漏掉 → 不对齐 → 割裂。

### 3.3 步骤⑤-b 4 路交叉回环检测 + SE(2) PGO (`scripts/pgo_fix.py`)
**回环相似度改用 4 路交叉最大**:
```
S[i,j] = max over (cᵢ,cⱼ)∈{1..4}²  desc[i,cᵢ] · desc[j,cⱼ]
```
即帧 i 的任一路与帧 j 的任一路只要相似就算同地点。实测帧 621↔501 从 **0.32 → 0.77**,反向回环被识别。实现上把 `(N,4,D)` 展平成 `(4N,D)` 算全相似度矩阵,再 `reshape(N,4,N,4).max(axis=(1,3))`。

**回环判定**:`sim > --sim(0.7)` 且 序号间隔 `> --gap(40)` 且 **当前建图空间距离 `< --dmax`**。

> ⚠️ **dmax 必须收紧(默认已改为 4m)**:4 路环视会让**矩形房间的两条对边**(相距 ~8m 但互相可见)的交叉相似度很高,若 `dmax` 太大(如12m)会把对边误判为回环、PGO 把对边拉合 → **房间被挤扁成楔形**。`dmax=4` 排除对边(8m),保留真回环(同位置 ~2.8m)。这是 4 路回环的**对边误判**陷阱,必须配紧 dmax。

**SE(2) 位姿图优化**:
- 把轨迹投到 SE(2)(x, y, 航向 ψ)。
- **里程计边**:相邻 + 窗内跳边(gap∈{1,2,3,5,8}),保形刚度。
- **回环边**:仅约束"同地点位置相同",不约束航向(走廊往返反向,航向差由里程计定),权重 `--wloop(3.0)`。
- `scipy.least_squares`(soft_l1 鲁棒核,稀疏雅可比,`max_nfev=800`)求解;固定 0 号位姿规约自由度。
- **z 拉平到 1.3m**(楼层平面假设),消除垂直弯曲;用校正后 (Δψ, x, y, z=1.3) 重构每帧 `T_wc`。
→ `trajectory_pgo.npz`。实测 200 个回环,**回环残余距离中位 1.19m → 0.32m**。

### 3.4 步骤⑤-c 写回位姿 (`scripts/apply_pgo.py`)
把 `trajectory_pgo.npz` 的校正位姿写回 `trajectory/db/query.npz`(按 frame_idx 映射);**首次运行备份原始为 `trajectory_orig.npz`**(供重优化)。

### 3.5 步骤⑥ 4 路环视融合点云 (`scripts/build_4cam_points.py`)
用**对齐后的位姿**重建 360° 度量点云:
- 每 `--stride(6)` 帧取一个关键帧。每关键帧:cam1 的时序邻帧(-2,0,+2,给基线)+ cam2/3/4 当帧(补 360°)一起送 VGGT。
- 以 cam1 当帧的已知世界位姿 `T_wc1` 为锚,地面 1.3m 定本窗尺度,把所有相机点投到世界系。
- 置信度过滤(`depth_conf ≥ 55` 分位),下采样。
- **产物** `global_points_4cam.npz`:`points (M,3)` + `kt (M,)`(每点来源关键帧时间,供割裂检测着色)。
- **注意 PGO 的副作用**:位姿对齐后,两趟的 VGGT 单帧噪声点也会重合、密度翻倍,反而更易触发判障 → 故占据生成端把判障密度阈值调高(见 §3.6)。

### 3.6 步骤⑦ 2D 占据栅格(最终导航地图)(`vgpnav/occupancy.py` + `scripts/export_web.py`)
**判障(`OccupancyGrid`)**:点按相对地面高度 `h=z-ground_z` 分类:
- `|h| < ground_band(0.15)` → 贴地**可通行**(值 1);
- `ground_band ≤ h ≤ ceil(1.3)` → 候选**障碍**。
- **稳健判障 `grid_robust(min_hits, min_vext)`**:某格成为障碍(值 2)需 **点密度 ≥ min_hits 且 垂直跨度 `hmax-hmin ≥ min_vext(0.3m)`**(真墙跨高度,悬浮噪声不跨)。

**`export_web.py` 里生成最终占据图的三道工序**:
1. `grid_robust(min_hits=6, min_vext=0.3)`:**密度阈值 6**(PGO 对齐后真墙点重合、密度高,提阈值滤掉对齐时重合的 VGGT 噪声;真墙保留)。
2. **形态学清理**:去掉面积 < 5 格的孤立障碍小簇(残余噪声)。
3. **轨迹走廊强制可通行(关键)**:机器人物理走过的轨迹周围半径 0.75m **无条件标可通行、覆盖虚构障碍**。
   - 依据:"机器人走过 ⇒ 必可通行"。DB 节点都在轨迹上 ⇒ **必可设起终点**。
   - 实测把"DB 节点落在最大可通行连通区"的可达率从 **21/209 → 209/209**。

> **设计取舍**:漂移割裂在占据层无法完美根除(单目重建噪声固有)。PGO 对齐"墙割裂"(墙线变清晰),`grid_robust(6)` 滤重合噪声,轨迹走廊兜底保证可达——三者叠加,得到一张**可设起终点、可 A\* 导航、不穿真墙**的占据地图。

## 4. 在线导航 —— 端到端单帧推理 (`vgpnav/pipeline.py` `VGPNav.run`)

输入 query 4 路环视(或单图),输出位姿 `T_wc` + 度量点云 + 局部占据图。

### 4.1 步骤① VPR 检索
4 路各算描述子,对 DB 取**最大相似** `sims = (q_descs @ db.desc.T).max(axis=0)`(任一路认出该地点即可,抗朝向差)。

### 4.2 步骤② 几何感知检索 Algorithm 1 (`vgpnav/retrieval.py`)
从 VPR top-N(`search_pool_N=40`) 中选 k=10 个**既相似又视角分散**的参考:
- **空间离群剔除**:以 VPR 最可信匹配为锚(前 `anchor_topm=5` 的几何中位,Weiszfeld 法),**半径门控** `radius_m=7` 内为有效候选(抗强感知混叠,办公室走廊重复纹理);若不足 k 个则补最近的。(论文版用 95 分位剔除,本复现改半径门控更稳。)
- **多样性选择(贪心)**:从 best 出发,每次选使 `J(c)=D_geo(c)·S_norm(c)` 最大的;`D_geo(c)=min_{已选s} (λ_pos·d_pos + λ_ang·d_ang)`,`λ_pos=0.2, λ_ang=0.6`。位置+朝向都分散 → 给 VGGT 大基线约束。

### 4.3 步骤③ VGGT 重建
`vggt.infer([query 4路] + k 个参考图)`,一次前向得所有帧的相对位姿 `extri`、稠密点 `world_points`、置信度。参考帧带已知度量位姿,提供约束。

### 4.4 步骤④ 加权运动平均定位 (`vgpnav/motion_averaging.py`)
两阶段解耦,鲁棒估计 query 世界位姿:
- **旋转共识**:每个参考给一个旋转假设 `R_hyp_i = R_wc_i · R_i · R_qᵀ`;四元数平均(Markley,Gram 矩阵最大特征向量)得初值,再用高斯核(基于 MAD)加权重平均 → `R_wc`。
- **射线三角化(IRLS)**:参考相机中心 `c_i` 出发、指向 query 的方向 `v_i`,解 `min_t Σ Ω_i ‖P_i(t-c_i)‖²`(`P_i=I-v_iv_iᵀ` 投影到垂直射线的平面),闭式 `t=A⁻¹b`。权重 `Ω_i = w_rot · w_geo · w_res`:`w_geo` 惩罚近平行射线(三角化病态),`w_res` 用 Tukey biweight(`c=4.685`)IRLS 迭代 10 次抑制离群。
→ `T_wc = [R_wc | t]`。

### 4.5 步骤⑤ 在线地面锚定尺度恢复 (`vgpnav/scale_recovery.py::anchor_query_points`)
query 的 VGGT 点云尺度模糊。用定位旋转 `R_wc` 把点转到世界朝向,沿重力轴 `find_ground_peak` 找本帧地面高度 `ground_z_rel`(应为负),`scale = 1.3 / (-ground_z_rel)`,把点放到世界系。维持同一支撑面上的度量尺度。

### 4.6 步骤⑥ 占据图 + A* (`vgpnav/occupancy.py` + `vgpnav/planner.py`)
- 多视角点云投世界系 → 以 query 为中心的局部 `OccupancyGrid`(`res=0.05m, range=8m`)。
- `astar`:8 连通,障碍按机器人半径膨胀,启发式 Chebyshev,得无碰撞路径。
- **耗时**(对应论文 Table IV,L40):VGGT 占大头(~0.4-0.8s),运动平均/地面锚定各几毫秒到几十毫秒。

> 注:web 可视化里的全局导航(`outputs/web/index.html` 的 JS A*)用的是**离线建好的全局占据图**(§3.6),并对**未知格**加通行代价(`UNK=4`),让路径**强偏好已知自由走廊、绕开未观测区(如玻璃会议室)**,避免穿越无观测区。

## 5. 本复现解决的关键工程问题(实战记录)

| 问题 | 现象 | 根因 | 解决 |
|---|---|---|---|
| **左下角虚构障碍淹没 DB 节点** | 房间内部被障碍填满,无法设起终点(可达 21/209) | 漂移割裂:绕行两趟墙错位堆叠 + 占据判障用单一 ground_z | `export_web` 轨迹走廊**强制可通行覆盖**(→209/209) |

## 6. 核心数学公式速查

| 模块 | 公式 | 含义 |
|---|---|---|
| 尺度锚定 | `scale = 1.3 / (cam_z - ground_z)`;`z_shift = -ground_z·scale` | 用相机离地 1.3m 定米制尺度,地面归零 |
| 4路回环 | `S[i,j] = max_{cᵢ,cⱼ} desc[i,cᵢ]·desc[j,cⱼ]` | 任一路对任一路最大相似(抗反向) |
| 旋转假设 | `R_hyp_i = R_wc_i · R_i · R_qᵀ` | 第 i 参考对 query 的世界旋转推测 |
| 射线三角化 | `min_t Σ Ω_i ‖(I-v_iv_iᵀ)(t-c_i)‖²` | 多参考射线最小二乘交点(query 位置) |
| 检索多样性 | `J(c)=D_geo(c)·S_norm(c)`,`D_geo=min_s(λ_pos d_pos+λ_ang d_ang)` | 既相似又分散 |
| 稳健判障 | `障碍 ⟺ 密度≥min_hits 且 (hmax-hmin)≥min_vext` | 真墙跨高度,噪声不跨 |
