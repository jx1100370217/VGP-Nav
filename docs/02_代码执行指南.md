## 1.5 多数据集机制(扩展数据集)

VGP-Nav 支持多个数据集,**加新数据集只需在 `vgpnav/datasets.py` 增加一条 `DatasetSpec`,不改任何脚本/核心代码**。

### 1.5.1 怎么切换数据集
所有脚本里 `cfg = Config()` 默认读环境变量 `VGPNAV_DATASET`(没有则 `Mapping_C8`):
```bash
export VGPNAV_DATASET=ChuangfuTower            # 之后所有脚本都对创富大厦生效
# 或单条命令前缀: VGPNAV_DATASET=ChuangfuTower $PY scripts/build_database.py
```
产物自动隔离到 `outputs/{数据集名}/`(db/query/web 互不干扰)。也可在代码里 `Config(dataset="名字")` 显式指定。

### 1.5.2 已注册数据集(`vgpnav/datasets.py`)
| 数据集 | 相机高度 | 4环视布局 (camera_1/2/3/4) | 数据 / 帧数 | 说明 |
|---|---|---|---|---|
| `Mapping_C8` | 1.3m | 前左/前右/后右/后左 | `memory-nav/Mapping_C8`, 626帧 | 小车环视 |
| `ChuangfuTower` | 1.70m | 左后/左前/右前/右后 | `data/ChuangfuTower/extracted_data`, 35帧 | 创富大厦头盔环视(身高173cm→相机高1.70m) |
| `Mappingdata_C7` | 1.3m | 前左/前右/后右/后左 | `data/Mappingdata_C7`, 390帧 | 小车环视(参数同 C8) |
| `Mappingdata_Firstfloor` | 1.3m | 前左/前右/后右/后左 | `data/Mappingdata_Firstfloor`, 284帧 | 小车环视(参数同 C8) |

### 1.5.3 加一个新数据集(3 步)
1. **准备图片**:4路环视命名 `{timestamp}_camera_{1..4}.jpg`,放到某目录(如 `data/新数据集/extracted_data`)。
2. **加 spec**:在 `vgpnav/datasets.py` 的 `DATASETS` 增加一条:
   ```python
   "新数据集名": DatasetSpec(
       name="新数据集名",
       data_dir="/abs/path/extracted_data",
       camera_height_m=1.7,            # 相机离地高度(米) —— 尺度锚, 最关键参数
       cam_layout=[("左前",2),("右前",3),("左后",1),("右后",4)],  # 4环视2x2: 左上,右上,左下,右下(前进方向在上排)
       cam_params="",                 # ""=复用 memory-nav/cam/params.yaml; 不同型号相机给自己的内参yaml
       map_stride=1, n_query=8),
   ```
3. **跑流程**:`export VGPNAV_DATASET=新数据集名` 后照 §0 / §3 跑(建图)、§4(导航测试)。

**关键说明**:
- `camera_height_m`:相机光学中心离地高度。机器人/头盔不同就改此值(尺度由它线性锚定,影响整图度量)。
  创富大厦是头戴头盔,主锚 camera_1 离地 ≈ 身高173cm − 头顶到头盔帽沿~12cm + 相机在帽沿上95mm ≈ 1.70m。
- `cam_layout`:决定 web 4环视 2×2 怎么摆(前进方向恒在上排)。按该数据集 4 相机的物理方位填,`index.html` 会动态生成。
- `cam_params`:同型号鱼眼可复用 C8 的(创富即如此,均 1920×1536 AVM 鱼眼);**不同相机必须给自己的 MEI 内参**(`[xi,fx,fy,cx,cy]+[k1,k2,p1,p2]`),否则去畸变错。
- ⚠️ **短序列**(如创富 35帧):VGGT 滑窗链式重建尺度漂移大、常无回环(PGO 只能 z 拉平、修不了 x-y),建图质量受限——**数据越长越完整越好**。

## 3. 建图流程逐步详解(原始数据 → 最终地图)

> 表格速览,逐条详见下文。"耗时"为 L40 参考值。

| # | 命令 | 作用 | 主要输入 | 主要输出 | 耗时 |
|---|---|---|---|---|---|
| ① | `build_database.py` | VGGT滑窗建库+地面锚定 | Mapping_C8 + cam params | trajectory/db/query/global_points.npz, meta.json | ~5-10min |
| ② | `compute_4cam_desc.py` | 4路环视VPR描述子 | Mapping_C8(4路) | allframe_desc_4cam.npy | ~3min |
| ③ | `pgo_fix.py` | 4路交叉回环PGO | trajectory_orig/trajectory + 4cam desc | trajectory_pgo.npz, pgo_before_after.png | ~1-15min* |
| ④ | `apply_pgo.py` | 写回校正位姿 | trajectory_pgo.npz | 覆盖 trajectory/db/query;备份 trajectory_orig | <1s |
| ⑤ | `build_4cam_points.py` | 360°融合点云重建 | trajectory(校正)+Mapping_C8 | global_points_4cam.npz | ~10-15min |
| ⑥ | `run_query.py` | 定位评测+局部占据+A* | db/query/global_points + query 4路图 | results.npz + 4张评测图 | ~3-5min |
| ⑦ | `export_cams.py` | 4路环视缩略图 | db.npz + Mapping_C8 | web/thumbs/n*_c*.jpg(836) | ~2-3min |
| ⑧ | `export_web.py` | 导出最终web | trajectory/4cam点云/db/query/results/node_names | web/{index.html,data.js} | ~1-2min |

\* PGO 耗时取决于 `max_nfev`(代码内 800)。scipy 数值雅可比单线程较慢;若改大到数千会跑十几分钟(见 §7)。

### ① `build_database.py` —— 建库
- **做什么**:`build_database()` 执行 VGGT 滑窗链式重建(两遍尺度)+ 全局重力对齐 + 地面 1.3m 锚定 + DB/query 划分 + DB 描述子;然后画 `database_overview.png`(俯视轨迹+侧视相机高度验证)。
- **关键 config**(`vgpnav/config.py`):`map_stride=1`(全帧)、`db_sub=3`、`n_query=25`、`chunk_size=10`、`chunk_overlap=6`、`camera_height_m=1.3`、`device`。
- **输出**:`outputs/db/{trajectory,db,query,global_points}.npz` + `meta.json`。

### ② `compute_4cam_desc.py` —— 4路描述子
- **做什么**:对 626 帧的 camera_1~4 去畸变后算 SelaVPR 描述子,存 `allframe_desc_4cam.npy (626,4,D)`。
- **为何需要**:回环检测要识别"反向经过同一地点"(单目漏检),4路交叉相似度能抓到(详见原理文档 §3.2)。
- **依赖**:① 的 `trajectory.npz`(确定帧集)。

### ③ `pgo_fix.py` —— 4路交叉回环 PGO
```bash
$PY scripts/pgo_fix.py --sim 0.70 --dmax 4 --gap 40 --wloop 3.0
```
- **参数**:
  - `--sim 0.70`:**近距离**回环的4路交叉相似度阈值(局部重访)。
  - `--dmax 4`:**近距离**回环的建图空间距离上限(米);此距离内 sim>--sim 即认回环。
  - `--simfar 0.85`(★分层回环关键):**远距离**(超 dmax)回环的高相似阈值,解决 dmax 两难——
    单靠距离门控时 `dmax` 小会漏**大漂移真闭环**(机器人绕一圈回起点,首尾本是同地点、但因累积漂移
    建图差几十米),`dmax` 大又误判**矩形对边**(~8m互相可见)。改为:近距离靠 sim+距离,远距离靠
    **高相似**(真闭环外观极似 0.90+,对边/巧合中等 <0.83)。实测 C7 首尾闭环(0.90-0.93)被捕获、
    #0-#375 建图 35m→2.8m;C8 远距离最高仅 0.717,故 simfar=0.85 对 C8 零触发(回测不变差)。
  - `--gap 40`:回环最小帧间隔。`--wloop 3.0`:回环边权重。`--topk 4`:每帧最多保留回环数。
- **输入**:优先 `trajectory_orig.npz`(若存在),否则 `trajectory.npz`;+ `allframe_desc_4cam.npy`。
- **输出**:`trajectory_pgo.npz`(校正位姿,z 拉平到 1.3m)+ `pgo_before_after.png`。
- ⚠️ **orig 陷阱**:`pgo_fix` 优先读 `trajectory_orig.npz`。若你**重新建库**后想重跑 PGO,旧的 `trajectory_orig`(上次建库的)会和新数据不一致 → 先 `cp outputs/db/trajectory.npz outputs/db/trajectory_orig.npz` 让 PGO 基于当前数据(首次全新流程无此问题,因 orig 尚不存在)。

### ④ `apply_pgo.py` —— 写回
- **做什么**:把 `trajectory_pgo.npz` 校正位姿按 frame_idx 写回 `trajectory/db/query.npz`;**首次运行备份 `trajectory_orig.npz`**。
- **验证**:`apply` 后可查同一物理点两趟是否对齐(如 #621 与 #501 的建图距离应从 ~2.8m 降到 ~0.2m)。

### ⑤ `build_4cam_points.py` —— 360°融合点云
```bash
$PY scripts/build_4cam_points.py --stride 6   # --limit N 仅重建前N关键帧并出测试图
```
- **做什么**:用**对齐后**的 cam1 位姿为锚,每 `stride` 帧把 cam1 邻帧+cam2/3/4 送 VGGT,地面 1.3m 定尺度,融合成 `global_points_4cam.npz`(`points`+`kt`)。
- **依赖**:④ 写回的校正 `trajectory.npz`。这是**占据地图的点云来源**。

### ⑥ `run_query.py` —— 定位评测(端到端 pipeline)
- **做什么**:对 25 个 held-out query 跑完整 `VGPNav.run`(VPR→检索→VGGT→运动平均→尺度→局部占据→A*),输出定位误差 + 4 张图。
- **输出**:`outputs/query/results.npz`(定位误差,供 ⑧ 在 web 上展示)+ `localization_overview/query_montages/astar_demo/global_occupancy.png`。

### ⑦ `export_cams.py` —— 4路环视缩略图
- **做什么**:为每个 DB 节点生成 cam1~4 去畸变缩略图 `web/thumbs/n{i}_c{c}.jpg`(836 张)。web 的**4环视机器人视图/DB详情**靠这些文件。

### ⑧ `export_web.py` —— 导出最终 web(含最终占据地图)
- **做什么**:读 4路点云 + 轨迹 + DB + query + (可选)results/node_names,**生成最终占据栅格**(`grid_robust(min_hits=6)` + 形态学清噪 + 轨迹走廊强制可通行),连同点云/DB/定位结果写入 `data.js`,拷贝 `index.html`。
- **输出**:`outputs/web/{index.html, data.js}`(thumbs 由 ⑦ 提供)。

## 4. 可视化导航测试

### 4.1 启动 web 服务(在 L40,Flask 后端 = 静态 + 建图控制 + DB管理)
```bash
PY=/home/ubuntu/miniconda3/envs/internvla/bin/python
$PY scripts/server.py        # 0.0.0.0:8199; 托管 outputs/ + 提供 /api/(建图流程控制、DB增删改查); 依赖 flask(已装)
# 纯查看也可用静态服务: $PY -m http.server 8199 --directory outputs  (但建图控制/DB管理需 server.py)
```
浏览器打开 `http://192.168.50.72:8199/` —— **多地图总应用**(`web/portal.html` → `outputs/index.html`):顶层三 tab(🏗️建图流程 / 🧭导航 / 📍DB管理),**每个 tab 内右上切换地图**(深港国际C8/C7/1楼、创富大厦28楼,动态读 `/api/datasets`)。新数据集建好(`export_web` 会拷 portal.html + nav.html)即自动出现在地图切换条。

也可直接访问单个数据集的纯导航页:`http://192.168.50.72:8199/{数据集名}/web/`(= `nav.html`,如 `/Mapping_C8/web/`)。
- ⚠️ **改了 data.js/index.html 后,浏览器需硬刷新 `Ctrl+Shift+R`**(http.server 不发禁缓存头,会强缓存 data.js)。
- 服务随终端关闭而停;长期挂可用 `tmux`/`nohup`/`systemd`。

### 4.2 总应用三大模块(顶层 tab 切换;每个 tab 内右上切换地图,当前 tab 随之刷新)
- **🏗️ 建图流程**:8 步 pipeline 卡片(①建库 … ⑧导出),每步状态灯(绿=完成/灰=未跑/黄=运行中);点卡片看该步**产物图**(轨迹概览 / PGO前后 / 占据图等),点"▶运行选中步骤"由后端 subprocess 跑对应建图脚本并**实时流式显示日志**,"▶一键全流程"顺序跑全部 → 从原始数据到导航地图每步可监控可控制。
- **📍 DB 节点管理**:节点表格(序号/帧号/名称/XY),支持**改名 / 删除 / 从轨迹帧添加**(增删改直接写 `db.npz`+`node_names.json`),点行看该节点 4 环视。⚠️ 增删改后到「建图流程」重跑 ⑧导出地图,导航端才生效。
- **🧭 导航演示**:iframe 嵌该地图的纯导航页(`nav.html`:canvas 地图 + 设起终点 A* + 4环视),操作见下。

#### 导航演示步骤
页面左侧是**地图**(点云按高度着色、占据栅格、轨迹、DB 节点),右侧是控制/视图面板:
1. **看图层**:右侧"图层"勾选 点云/轨迹/DB节点/占据(障碍)/定位评测。
2. **设起终点 → 导航**:
   - 点 `① 设起点` → 在地图上单击一处(会吸附到最近的可通行 DB 走廊);
   - 点 `② 设终点` → 单击另一处;
   - 点 `▶ 开始导航` → JS A* 规划路径,机器人沿路径行进。
   - 遥测面板显示:路径长度、路点数、规划耗时、进度、当前位置(DB 地点名)等。
3. **机器人视图 · 4环视**:导航中实时显示当前位置的 **4 路环视**(2×2,前左①/前右②/后左④/后右③,前进方向恒在上),消除单视角倒走感。

## 7. 常见问题与踩坑(实战)

1. **PGO 跑很久不结束**:`pgo_fix.py` 用 scipy `least_squares` + **数值雅可比 + 单线程**,`max_nfev` 设大(如4000)会跑十几分钟。代码内已设 **800**(足够收敛且快);右上挤扁是 `dmax` 问题(对边误回环),不是收敛问题,别盲目加大迭代。
2. **重新建库后 PGO 用了旧轨迹**:`pgo_fix` 优先读 `trajectory_orig.npz`(可能是上次建库的)。重建后想重跑 PGO,先 `cp outputs/db/trajectory.npz outputs/db/trajectory_orig.npz`。
3. **web 改了不生效**:浏览器对同名 `data.js` 强缓存,**硬刷新 `Ctrl+Shift+R`**;或起服务时发 `Cache-Control: no-store` 头。
4. **DB 节点无法设起终点/被障碍淹没**:确认 `export_web.py` 的"轨迹走廊强制可通行"已生效(日志会打印"覆盖虚构障碍 N 格");可达率应是 209/209。
5. **房间被挤扁 / 首尾该闭合却没闭**:回环检测分层——近距离靠 `--dmax 4`(太大则对边误判致挤扁),远距离大漂移真闭环靠 `--simfar 0.85`(太大致漏闭环、太小引对边;C8 安全线 ~0.85)。C7 首尾闭环(建图差35m)靠 simfar 修复(#0-#375→2.8m);**单程无重访的数据(如 Firstfloor)本就无闭环可修**,尺度漂移得靠数据质量/VGGT 改善。
6. **占据图障碍偏多(PGO后)**:位姿对齐使噪声点重合,`export_web` 的 `grid_robust(min_hits=6)` 已配套滤噪;可视情况调 `min_hits`。
7. **穿玻璃会议室**:玻璃被动视觉重建不出,占据层无障碍;`index.html` A* 已用 `UNK=4` 未知格代价规避,根治需更稠密重建/主动深度传感。
8. **L40→GitHub push 失败**:`https` 偶发 TLS 握手失败,`git config http.version HTTP/1.1` 并重试可解决;需有写权限的 token。
